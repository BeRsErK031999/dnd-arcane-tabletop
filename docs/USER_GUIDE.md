# D&D Arcane Tabletop User Guide

## Назначение

D&D Arcane Tabletop — локальное Windows desktop-приложение для мастера. Кампании, сцены, заметки, ассеты, карточки, туман войны, инициатива и состояние экрана игроков хранятся локально.

Приложение не требует аккаунта, backend, облака или сетевого подключения игроков.

## Запуск в разработке

```bash
npm install
npm run dev
```

`npm run dev` запускает renderer и Electron master window. Для проверки только renderer route можно использовать:

```bash
npm run dev:renderer
```

После этого доступны:

- `http://127.0.0.1:5173/?screen=master`
- `http://127.0.0.1:5173/?screen=player`

## Кампании и сохранение

1. На стартовом экране создайте новый проект или запустите существующий из библиотеки.
2. Изменения кампании сохраняются в JSON.
3. Master UI показывает состояние autosave: ожидание, изменения, сохранение, сохранено или ошибка.
4. После изменений запускается debounced autosave.
5. При закрытии renderer выполняется финальный flush текущей кампании.

В редакторе кнопка `Проекты` в левом верхнем углу сохраняет изменения и возвращает в библиотеку. Повторный запуск проекта восстанавливает выбранную кампанию и активную сцену. Создание, импорт, экспорт и удаление проектов выполняются на стартовом экране, поэтому внутри рабочей области нет второго набора project CRUD.

По умолчанию в development JSON-файлы лежат в `data/campaigns`, а в packaged build - внутри Electron `userData`. В блоке `Папка проекта` можно открыть другую локальную папку проекта или сохранить текущую кампанию в выбранную папку. Backup-копии хранятся в `.backups` внутри активной папки проекта и ротируются до двух последних версий.

## Импорт и экспорт проекта

На стартовом экране доступны два desktop-действия:

- `Экспорт проекта` сначала показывает состав автономного файла `campaign.arcane-campaign`, а после подтверждения сохраняет используемые ассеты и материалы с флагом `Всегда добавлять`;
- `Импорт проекта` полностью проверяет пакет, добавляет файлы в управляемое SHA-256-хранилище без дублей и выбирает импортированную кампанию.

Перед экспортом текущая кампания сохраняется. Preview показывает число логических ассетов, уникальных blob, общий объём, используемые, дополнительно добавленные и встроенные материалы. Если кампания или исходный файл изменились после preview, приложение попросит сформировать состав заново.

Формат версии 2 хранит каждый уникальный SHA-256 один раз, не переносит absolute source paths и защищает каждый бинарный blob checksum. При конфликте campaign id импорт создаёт новый внутренний id и не перезаписывает существующую кампанию. Повреждённый пакет, небезопасный путь или несовместимая версия отклоняются до публикации проекта. Пакеты версии 1 продолжают импортироваться и при импорте мигрируют свои ассеты в managed store.

## Undo/Redo

Undo/Redo работает для основных изменений выбранной кампании: сцены, canvas, туман войны, ассеты, заметки, карточки, инициатива и player screen state.

- `Ctrl+Z` — отменить последнее campaign-действие.
- `Ctrl+Y` или `Ctrl+Shift+Z` — повторить отмененное действие.
- `Ctrl+S` — сохранить открытую кампанию.

Если фокус находится внутри поля ввода, `Ctrl+Z` остается текстовым undo поля.

## Экран игроков

Master window управляет отдельным player window через typed IPC.

В панели `Экран игроков` можно:

- открыть или сфокусировать player window;
- закрыть player window;
- включить или выключить fullscreen;
- отправить тестовую сцену или изображение;
- скрыть и снова показать экран игроков;
- сбросить player screen state.

Player window только отображает полученный `PlayerScreenState` и не содержит инструментов мастера.

## Сцены, canvas и туман войны

Сцена всегда остаётся центральной рабочей областью. В верхней полосе можно создать новую сцену и переключиться между существующими. Кнопка `Материалы` скрывает или показывает правую панель сцены, освобождая место для canvas.

Компактная навигация открывает `Ассеты`, `Бой`, `Заметки` и `Игроки` поверх сцены. Повторное нажатие активного пункта или `Escape` закрывает панель и возвращает фокус на `Сцены`. На узком экране навигация располагается горизонтально сверху, а функциональная панель занимает доступную ширину.

Рабочая область сцены поддерживает:

- сетку;
- viewport;
- измерения;
- объекты сцены;
- master-only объекты;
- туман войны;
- player-safe projection.

## Ассеты и handouts

В функциональной панели `Ассеты` можно подключать индексируемые папки, искать библиотечные изображения, импортировать выбранные файлы, редактировать теги, привязать карту к активной сцене или отправить изображение игрокам. Встроенный блок `Материалы` остаётся быстрым доступом к ассетам, персонажам и заметкам текущей сцены.

В панели `Заметки` можно вести master-only заметки и публичные handouts. Секретные заметки не отправляются игрокам.

## Инициатива

Combat tracker поддерживает ручное ведение инициативы:

- участники;
- порядок хода;
- старт/стоп;
- следующий ход;
- следующий раунд;
- переключатель видимости инициативы игрокам.

Player screen получает только public initiative projection без master-only ссылок, HP, AC и приватных заметок.

## Сборка Windows installer

Перед сборкой installer выполните проверки:

```bash
npm run lint
npm run typecheck
npm run test
npm run build
```

Сборка installer:

```bash
npm run dist:win
```

Артефакты создаются в `release/`. Эта папка не коммитится.
