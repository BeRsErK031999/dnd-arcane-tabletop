# D&D Arcane Tabletop User Guide

## Назначение

D&D Arcane Tabletop — локальное Windows desktop-приложение для мастера. Кампании, сцены, заметки, ассеты, карточки, туман войны, инициатива и состояние экрана игроков хранятся локально.

Приложение не требует аккаунта, backend, облака или сетевого подключения игроков.

## Запуск в разработке

```bash
npm install
npm run dev
```

`npm run dev` запускает renderer и Electron master window. Для проверки только renderer route можно использовать:

```bash
npm run dev:renderer
```

После этого доступны:

- `http://127.0.0.1:5173/?screen=master`
- `http://127.0.0.1:5173/?screen=player`

## Кампании и сохранение

1. В блоке `Кампании` создайте новую кампанию или откройте существующую.
2. Изменения кампании сохраняются в JSON.
3. Master UI показывает состояние autosave: ожидание, изменения, сохранение, сохранено или ошибка.
4. После изменений запускается debounced autosave.
5. При закрытии renderer выполняется финальный flush текущей кампании.

По умолчанию в development JSON-файлы лежат в `data/campaigns`, а в packaged build - внутри Electron `userData`. В блоке `Папка проекта` можно открыть другую локальную папку проекта или сохранить текущую кампанию в выбранную папку. Backup-копии хранятся в `.backups` внутри активной папки проекта и ротируются до двух последних версий.

## Импорт и экспорт проекта

На стартовом экране доступны два desktop-действия:

- `Экспорт проекта` сохраняет выбранную кампанию в автономный файл `campaign.arcane-campaign` вместе с прикреплёнными локальными ассетами;
- `Импорт проекта` проверяет пакет, копирует ассеты в локальное хранилище и выбирает импортированную кампанию.

Перед экспортом текущая кампания сохраняется. Каждый бинарный ассет защищён SHA-256 checksum. Если campaign id уже занят, импорт создаёт новый внутренний id и не перезаписывает существующую кампанию. Повреждённый пакет, несовместимая версия или отсутствующий локальный файл показываются как ошибка и не должны оставлять частично импортированный проект.

Версия формата 1 включает все прикреплённые к кампании ассеты. Будущая гибридная библиотека будет автоматически собирать реально используемые ассеты и отдельно отмеченные пользователем дополнительные файлы, сохраняя расширение `.arcane-campaign`.

## Undo/Redo

Undo/Redo работает для основных изменений выбранной кампании: сцены, canvas, туман войны, ассеты, заметки, карточки, инициатива и player screen state.

- `Ctrl+Z` — отменить последнее campaign-действие.
- `Ctrl+Y` или `Ctrl+Shift+Z` — повторить отмененное действие.
- `Ctrl+S` — сохранить открытую кампанию.

Если фокус находится внутри поля ввода, `Ctrl+Z` остается текстовым undo поля.

## Экран игроков

Master window управляет отдельным player window через typed IPC.

В панели `Экран игроков` можно:

- открыть или сфокусировать player window;
- закрыть player window;
- включить или выключить fullscreen;
- отправить тестовую сцену или изображение;
- скрыть и снова показать экран игроков;
- сбросить player screen state.

Player window только отображает полученный `PlayerScreenState` и не содержит инструментов мастера.

## Сцены, canvas и туман войны

В верхней полосе создаются и выбираются сцены. Рабочая область сцены поддерживает:

- сетку;
- viewport;
- измерения;
- объекты сцены;
- master-only объекты;
- туман войны;
- player-safe projection.

## Ассеты и handouts

В правой панели `Ассеты` можно импортировать изображения, редактировать теги, привязать карту к активной сцене или отправить изображение игрокам.

В панели `Заметки` можно вести master-only заметки и публичные handouts. Секретные заметки не отправляются игрокам.

## Инициатива

Combat tracker поддерживает ручное ведение инициативы:

- участники;
- порядок хода;
- старт/стоп;
- следующий ход;
- следующий раунд;
- переключатель видимости инициативы игрокам.

Player screen получает только public initiative projection без master-only ссылок, HP, AC и приватных заметок.

## Сборка Windows installer

Перед сборкой installer выполните проверки:

```bash
npm run lint
npm run typecheck
npm run test
npm run build
```

Сборка installer:

```bash
npm run dist:win
```

Артефакты создаются в `release/`. Эта папка не коммитится.
